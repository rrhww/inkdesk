# 网页前端说明

Inkdesk 的 Next.js 前端应用，当前服务于单人私有、Vault-first 的 AI 研发自动化控制台。

当前代码仍以 Ask 工作区为 `/app` 主入口；产品设计已经调整为 Dev Run-first，下一步应把 `/app` 改为研发任务台，并把 Ask 保留为 Context Ask。

## 当前页面模型

- `/`：直接重定向到 `/app`
- `/app`：当前为 Ask 工作区；目标改为 Dev Run Console
- `/app/raw`：raw 原始材料 vault
- `/app/ingest`：AI 提案审阅队列
- `/app/wiki`：已接受的 wiki 页面
- `/app/wiki/[id]`：单个 wiki 页面详情
- `/app/ask`：Context Ask，当前复用 `/app` 的 Ask 工作区

旧兼容路由已移除：`/app/inbox`、`/app/sources`、`/app/review`、`/app/topics` 不再作为产品入口。

## 入口规则

- 访问 `/`：跳转到 `/app`
- `/app/*` 无需登录即可访问

## 当前产品边界

- 没有公开阅读面
- 没有 plans 模块
- 没有 publish 模块
- 没有旧的 note editor 工作流
- 当前实现语言仍是 ask / raw / ingest / wiki，legacy note 只作为迁移来源存在
- 目标产品语言应扩展为 run / context ask / raw / ingest / wiki / skills / evals

## 本地启动

1. 在 `web/` 目录安装依赖：`npm install`
2. 复制 `web/.env.local.example` 为 `web/.env.local`
3. 如需接入真实后端，设置 `INKDESK_API_BASE_URL` 或 `NEXT_PUBLIC_API_BASE_URL`
4. 启动开发环境：`npm run dev`
5. 浏览器访问：`http://localhost:3000`

## 数据来源

默认情况下，前端会回退到本地 research fixtures。

如果配置以下任一变量，前端会优先读取 Python 主后端：

- `NEXT_PUBLIC_API_BASE_URL`
- `INKDESK_API_BASE_URL`

变量既可以写成 `http://localhost:8080`，也可以直接写成 `http://localhost:8080/api`。

当前接入后端的页面与动作包括：

- `/app`
- `/app/raw`
- `/app/ingest`
- `/app/wiki`
- `/app/wiki/[id]`
- `/app/ask`

这些页面在服务端渲染时直接访问后端接口。未配置后端地址时，会自动回退到本地 research fixtures。

## 测试

```powershell
npm test
npm run typecheck
npm run lint
npm run build
npm run e2e
```

如果本地 `PostgreSQL + MinIO + Python 主后端` 已启动，可以额外执行真实后端闭环验收：

```powershell
npm run e2e:fullstack
```

这个命令会验证无登录入口、Dev Run Console 以及 raw -> ingest -> wiki -> ask 的研究闭环。

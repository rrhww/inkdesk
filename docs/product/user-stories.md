# 用户故事

## 1. 主人登录与主系统入口

### 1.1 通过隐藏入口进入主系统

作为主人，我希望通过不公开的 `/login` 进入主系统，这样访客不会在公开输出页看到任何私有入口。

验收标准：

- 公共首页不出现登录入口
- 访问 `/login` 可进入登录页
- 登录成功后进入 `/app`
- 已登录后访问 `/` 会自动进入 `/app`

## 2. Agent 控制台

### 2.1 把 Agent 作为第一屏

作为主人，我希望登录后先看到 Agent 控制台，而不是普通列表页，这样系统能先帮我整理今天最重要的上下文。

验收标准：

- `/app` 首页标题与结构体现 Agent 控制台
- 首页可看到 Agent 建议
- 首页同时能通往任务计划与笔记模块

## 3. 笔记与知识库

### 3.1 维护知识资产

作为主人，我希望持续记录和整理自己的笔记，这样主系统的判断与检索都有稳定事实来源。

验收标准：

- 可以查看所有笔记
- 可以进入单篇笔记编辑页
- 可以通过标签和摘要识别内容归属

### 3.2 让知识资产服务于主系统

作为主人，我希望笔记不只是文档存放区，而是整个系统的知识底座。

验收标准：

- 笔记文案以“知识资产”表述
- 编辑页能看到发布状态与公开输出状态

## 4. 任务 / 计划

### 4.1 推进当前事项

作为主人，我希望有一个任务 / 计划页面来组织正在推进的事情，这样主系统不只是记录，还能帮我推动进度。

验收标准：

- `/app/plans` 可访问
- 至少展示今日任务与进行中计划
- 任务页面语义与主系统整体一致

## 5. 全局检索 / 知识召回

### 5.1 重新召回上下文

作为主人，我希望通过检索快速回到相关笔记和历史判断，这样 Agent 与我自己都能重新获得上下文。

验收标准：

- 可以按关键词搜索笔记
- 搜索页文案体现“召回上下文”而不是单纯文件检索

## 6. 公开输出与公开阅读

### 6.1 向访客展示公开内容

作为访客，我希望在 `/` 看到作者介绍、公开文章和项目链接，这样我能快速理解作者与公开内容。

验收标准：

- `/` 只展示公开输出信息
- 首页包含作者介绍、公开文章、项目/链接
- 页面中没有主系统入口

### 6.2 阅读公开文章

作为访客，我希望打开公开文章详情页阅读完整内容，这样公开输出层能够承担稳定的公开表达职责。

验收标准：

- 公开文章可从首页进入
- 文章详情页可正常阅读
- 详情页可以返回公开输出首页

## 7. 次级发布模块

### 7.1 从主系统发布到公开输出层

作为主人，我希望把部分内容从主系统发布到公开输出层，但不希望发布模块继续主导整个产品。

验收标准：

- `/app/publish` 继续存在
- 页面语义是“把内容发布到公开输出”
- 发布模块在导航中处于次级位置

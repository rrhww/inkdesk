# 用户故事

## 1. 主人登录与主入口

### 1.1 通过隐藏入口进入系统

作为主人，我希望通过不公开的 `/login` 进入系统，这样研究工作区只服务我自己。

验收标准：

- 访问 `/login` 可进入登录页
- 登录成功后进入 `/app`
- 已登录后访问 `/` 会自动进入 `/app`
- 未登录访问 `/app/*` 会自动跳转 `/login`

## 2. 问答作为知识体检

### 2.1 直接发起研究问答

作为主人，我希望登录后立刻进入问答型研究界面，这样我可以直接测试现有知识是否足够回答当前问题。

验收标准：

- `/app` 第一感知是问答入口
- 可以直接输入问题
- 回答中能看到引用来源、知识缺口和后续追问建议

### 2.2 问答输出健康信号

作为主人，我希望 Ask 不只给出答案，也告诉我这次回答暴露了哪些知识问题，这样我知道系统哪里还需要修。

验收标准：

- Ask 回答展示 knowledge gaps
- Ask 回答能说明使用了哪些 wiki / raw
- Ask 能提示是否存在值得沉淀的 writeback candidate

### 2.3 继续追问并继承上下文

作为主人，我希望继续追问时能继承上一轮研究上下文，这样系统不会每次都从零开始。

验收标准：

- follow-up 会携带上一轮 Ask 上下文
- 页面能提示当前正在延续上一轮问答
- 新回答会复用上一轮相关证据和缺口

## 3. 原始材料与 raw backlog

### 3.1 导入研究材料到 raw

作为主人，我希望把网页、PDF 和文本材料先导入 raw，这样所有后续知识沉淀都有统一入口。

验收标准：

- 可以导入网页
- 可以导入 PDF
- 可以粘贴文本材料
- 导入后材料进入 raw 列表，并保留 locator / vaultPath 等信息

### 3.2 看到尚未消化的材料

作为主人，我希望看到哪些 raw 材料还没有被有效编译，这样我知道知识系统还有哪些输入没有被消化。

验收标准：

- 系统能展示 raw backlog 信号
- raw 页面能区分已进入提案、已沉淀和仍待处理的材料

## 4. 审阅认知变更

### 4.1 审阅 AI 编译结果

作为主人，我希望先看到 AI 给出的提案，再决定是否写入 wiki，这样正式知识不会被静默篡改。

验收标准：

- `/app/ingest` 可访问
- 每条提案能看到 topic 归属、解释、claims、证据和开放问题
- 可以接受或忽略提案

### 4.2 区分提案类型

作为主人，我希望知道一条提案是新知识、旧知识补丁、开放问题，还是低证据推测，这样我能更快做出审阅判断。

验收标准：

- 提案能显示它的认知变更类型
- 提案能关联来源、Ask 或 health signal
- 低证据内容不会伪装成正式知识

## 5. 知识库与 claim

### 5.1 浏览沉淀后的当前理解

作为主人，我希望在知识库中浏览已经确认的当前理解，这样我能把正式知识和原始材料区分开。

验收标准：

- `/app/wiki` 可访问
- 可以查看 wiki 列表
- 页面能显示来源数量、开放问题数量和健康摘要

### 5.2 查看单篇 wiki 的结构化知识

作为主人，我希望打开单个 wiki 页面时，看到当前理解、claims、开放问题和来源，这样它能作为后续研究的稳定上下文。

验收标准：

- 可以进入单个 wiki 页面详情
- 页面结构体现 understanding、claims、questions 和 sources
- 页面能暴露仍未解决的问题

## 6. Ask 回写

### 6.1 从 Ask 生成新的知识提案

作为主人，我希望在一轮 Ask 后把有价值的回答沉淀回知识库，但仍然经过审阅闸门。

验收标准：

- Ask 页有“沉淀到 wiki”动作
- 点击后会生成 ingest 提案，而不是直接写 wiki
- 若本轮使用了外部网页，会先把这些来源保存到 raw

## 7. 知识健康总览

### 7.1 查看最小健康状态

作为主人，我希望看到系统当前有哪些知识健康问题，这样我能优先处理最值得修的地方。

验收标准：

- 系统至少展示 raw backlog、review backlog、open questions 三类信号
- 健康信号能跳转到对应页面或动作
- 健康信号不会被包装成复杂评分，而是清楚说明问题是什么

## 8. 联网补料

### 8.1 显式联网，而不是默认联网

作为主人，我希望联网补料是一个显式动作，而不是默认增强，这样我能控制外部资料何时进入研究过程。

验收标准：

- Ask 默认是纯 vault 模式
- 只有切换到 `vault_plus_web` 时才允许联网补料
- 页面明确提示外部来源尚未进入 vault

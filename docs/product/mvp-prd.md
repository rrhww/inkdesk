# Inkdesk MVP 产品需求文档

## 1. 产品背景

Inkdesk 被定义为一个会长期使用的超级个人工作台。

这意味着：

- 它必须同时服务记录、判断、推进与公开输出
- 它不能只是存笔记，还要成为主人进入系统后的工作中枢
- 它要能容纳 Agent、任务计划和未来自动化能力，但 MVP 不能一次做满

## 2. 产品定位

Inkdesk 的官方主定位是：

- 一个围绕长期项目运转的个人主系统

MVP 阶段的当前实现形态是：

- 一个由 `个人笔记系统 + Agent 中枢 + 任务/计划系统` 组成的私有主系统
- 一个对外承接 `公开文章 + 项目分享` 的公开输出层

这意味着 Inkdesk 当前 MVP 采用“主系统 + 输出层”的结构：

- 对外：访客看到从主系统派生出来的公开文章与项目分享
- 对内：主人登录后进入自己的主系统

## 3. 目标用户

### 主要用户

- 主人，也就是你自己
- 使用目标：进入主系统，借助 Agent、笔记与任务计划组织自己的长期工作

### 次要用户

- 访问公开页面的读者
- 使用目标：阅读你公开的文章，理解你的作者身份与项目方向

## 4. MVP 目标

MVP 要先跑通两条链路：

### 链路一：主人主系统链路

1. 通过隐藏入口登录
2. 进入 Agent 控制台
3. 在笔记与知识库中记录和整理内容
4. 在任务/计划模块中推进正在发生的事情
5. 通过检索快速召回历史上下文

### 链路二：公开输出链路

1. 访客访问 `/`
2. 看到作者介绍、公开文章和项目/链接分享
3. 打开公开文章详情页阅读

如果这两条链路都成立，MVP 就具备了“主系统骨架 + 输出层”的最小闭环。

## 5. 核心价值

- 对主人来说：不再只是存文档，而是拥有一个能组织知识、任务与上下文的主系统
- 对长期系统来说：笔记继续作为底层事实来源，Agent 与任务系统围绕它工作
- 对公共表达来说：可以稳定地把主系统中的成熟内容发布到公开输出层
- 对未来演进来说：为更深的 Agent 协同、自动化编排和超级个人体保留空间

## 6. MVP 范围内

### 主人入口与私有系统

- 不公开的 `/login`
- 单主人登录
- `/app` 私有主系统入口
- 根路由根据登录态做公开输出入口 / 主系统切换

### 主系统核心模块

- Agent 控制台
- 笔记与知识库
- 任务 / 计划
- 全局检索 / 知识召回
- 知识资产编辑器

### 公开输出 / 分享层

- 作者介绍
- 公开文章列表
- 项目 / 链接入口
- 公开文章详情页

### 次级模块

- 将部分内容从主系统发布到公开输出层
- 管理已公开与未公开内容

## 7. MVP 范围外

- 多人协作
- 评论、点赞、关注
- 实时协同编辑
- 完整工作流自动化编排
- 多角色权限体系
- 独立桌面客户端
- 真正的自主 Agent 执行系统
- 面向公众的产品官网式营销站

## 8. 核心流程

### 流程一：主人进入主系统

1. 直接访问 `/login`
2. 登录后进入 `/app`
3. 首页先看到 Agent 控制台
4. 再进入笔记、计划和检索模块

### 流程二：知识资产沉淀

1. 在主系统中创建或打开笔记
2. 编辑内容并手动保存到主系统
3. 通过标签与检索重新召回
4. 让 Agent 基于这些内容给出建议

### 流程三：任务推进

1. 在任务 / 计划模块记录事项
2. 将事项与笔记上下文连接
3. 由 Agent 协助拆解下一步动作

### 流程四：公共输出

1. 从主系统中挑选成熟内容
2. 发布到公开输出层
3. 在 `/` 与 `/articles/[slug]` 被访客阅读

## 9. 产品原则

- 主系统优先：先保证你自己每天愿意进入和使用
- 边界清晰：公开输出层不暴露主系统入口，主系统也不为访客而设计
- 笔记为底：内容事实来源必须持续落回笔记系统
- Agent 为中枢：主页优先服务判断与组织，而不只是展示列表
- 任务可推进：系统要帮助事情往前走，而不只是沉淀
- 低复杂度优先：MVP 先搭产品骨架，不虚构完整智能能力

## 10. 成功标准

MVP 达成以下条件即可视为成功：

- 访客访问 `/` 时只能看到公开输出页
- 主人通过 `/login` 能进入主系统
- 登录后访问 `/` 会自动进入 `/app`
- `/app` 第一屏已成为 Agent 控制台
- 笔记、任务计划、检索三个模块能在主系统中形成基本协同
- 至少一篇内容能稳定从主系统发布到公开输出层

## 11. 阶段边界

### MVP 完成标准

- 完成主系统骨架与输出层入口
- 完成 Agent 控制台、笔记系统、任务计划的最小产品模型
- 保留公开输出能力，但不把它当作主产品轴心

### MVP 之后的演进

- 第一优先：增强 Agent 协同深度
- 第二优先：增强任务与计划的执行能力
- 第三优先：引入自动化与工作流
- 最终目标：演进为真正的超级个人工作台

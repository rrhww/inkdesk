# 产品路线图

## 目标

这份路线图说明 Inkdesk 如何从当前 MVP 演进为长期使用的超级个人工作台。

## 阶段一：主系统骨架 + 输出层 MVP

### 阶段目标

- 建立公开输出入口与主系统的清晰边界
- 建立主人登录后的 Agent 控制台
- 建立笔记系统、任务计划和检索的主系统骨架

### 核心能力

- 公开输出首页
- 公开文章详情页
- 隐藏登录入口
- Agent 控制台
- 笔记与知识库
- 任务 / 计划
- 全局检索
- 次级发布模块

### 完成标志

- 主人与访客看到的是两套职责清晰的访问面
- 主系统已经具备可持续进入和使用的入口感

## 阶段二：Agent 协同增强

### 阶段目标

- 让 Agent 从“首页入口”升级为“真正参与判断与组织工作的中枢”

### 核心能力

- 最近上下文汇总
- 基于笔记的回顾与提炼
- 基于任务的下一步建议
- 跨笔记的主题归纳

### 进入条件

- 主系统首页结构稳定
- 笔记与计划数据已经形成稳定输入

## 阶段三：任务与计划深化

### 阶段目标

- 让主系统真正具备推进事情的能力，而不只是展示事项列表

### 核心能力

- 任务状态流转
- 计划拆解
- 截止时间与优先级
- 任务与笔记的关联

## 阶段四：自动化与工作流

### 阶段目标

- 引入自动化执行，让主系统具备更强的编排能力

### 核心能力

- webhook / 定时任务
- 自动摘要
- 计划提醒
- 与外部工具的连接

### 推荐策略

- 优先接入成熟工具，不在早期自研复杂引擎

## 阶段五：超级个人工作台

### 阶段目标

- 将笔记、Agent、任务计划、自动化和公共表达整合成统一使用平台

### 核心能力

- 一个真正统一的主系统首页
- 多时间尺度的工作视图
- 更强的上下文记忆
- 更稳定的个人长期运营能力

## 演进原则

- 先把主系统做顺，再增强智能层
- 先把主人体验做扎实，再考虑更大叙事
- 公开输出层永远服务于公开表达，不承担主系统入口职责
- 所有增强能力都必须和笔记系统兼容

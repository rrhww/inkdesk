# 产品定位与形态

> 基于文章《AI研发自动化：Wiki知识库+技能包》的思考和对 Inkdesk 产品方向的重新确认。

## 产品定位

```text
Inkdesk 是一个个人 AI 知识工坊。

它把 LLM-Wiki 知识库、Skill 技能包和评测系统整合成一个完整的应用。

你用 Inkdesk 来：
  - 积累知识（导入材料 → AI 编译 → 持续生长）
  - 使用知识（提问 → 引用 Wiki → 沉淀新发现）
  - 自动化工作（Skills 执行重复任务）
  - 看到成长（评测系统告诉你知识库在变好）

如果你也用 Claude Code 或 Codex，
Inkdesk 通过 MCP 接入它们，作为统一的知识底座。
但 Inkdesk 本身是一个独立、完整的产品。
```

## 产品形态

### 界面结构

```text
/app
├── ask/           主工作区 —— 提问、追问、沉淀
├── review/        审阅台 —— 待处理的 AI 提案（接受/拒绝/修改）
├── wiki/          知识库浏览 —— 页面、关联图、来源追溯
├── skills/        技能工作台 —— 浏览、运行、审阅输出
├── health/        健康仪表盘 —— 问题列表、质量趋势
├── evals/         评测中心 —— 跑分、版本对比、改进建议
└── settings/      基础设置
```

### 核心闭环

```text
raw → ingest → wiki → ask → 沉淀 → ingest
```

Ask 不只是问答，而是知识生产的起点：

```text
提问 → 基于 Wiki 回答（带来源引用）
     → 追问（上下文自动关联）
     → 选中有价值片段 → 一键沉淀
     → 后台拆解 → 生成 Wiki patch 提案
     → 进入审阅队列
```

### 三层扩展

```text
schema → skills → evals → runs → harness
```

- `schema/`：面向 Agent 的知识维护规则
- `skills/`：可复用工作流说明
- `evals/`：golden tasks、rubrics 和评测运行记录
- `runs/`：Agent 运行记录、输出和审阅状态
- `harness`：门禁、重试和回滚编排

### MCP Server 是产品的延伸

```text
人在 Inkdesk Web → 提问、审阅、管理
人在 Claude Code  → 通过 MCP 取上下文、提交沉淀 → 回到 Inkdesk 审阅
```

两条路径共享同一个 vault 和同一个审阅队列。MCP 是产品的接入方式之一，不是替代品。

## 与文章方案的对比

| | 文章方案（工具拼装） | Inkdesk（产品） |
|------|------|------|
| 知识摄入 | Claude Code 对话中执行 skill | Web 引导式上传 → 后台编译 → 审阅队列 |
| 审阅提案 | 对话中逐条确认 | 独立审阅台，diff 预览，批量操作 |
| 知识浏览 | Obsidian | Web 内浏览 + 关联图 + 来源追溯 |
| 搜索 | qmd CLI | 内置搜索，搜索即对话 |
| 健康检查 | 手动跑 lint | 自动定期扫描 + 仪表盘 + 趋势图 |
| 评测 | `claude -p` 子进程 | 评测中心：配置 golden tasks → 跑分 → 对比版本 |
| 多平台 | 手动配置各平台 | 开箱即用，MCP 作为可选扩展 |
| 上手 | 需学会 Obsidian + qmd + Git + Claude Code | 打开浏览器，登录，开始 |

## 从文章吸收的结构改进

| 改进点 | 说明 |
|--------|------|
| Schema 提前 | Schema 在内容灌入前就位，不做事后补救 |
| index.md + log.md | 初始化就生成全局导航和操作日志 |
| KB-META.md | 集中管理元信息，避免路径硬编码 |
| status 字段 | 页面 frontmatter 从初始化就包含 stub/draft/stable/outdated |
| 评测分层 | 结构层 → 实用层 → 信号层 → 治理层 |
| 盲评机制 | 评测时分离生成和评分，避免自我偏好 |

## 当前开发策略

**先打造 MCP Server，再逐步打磨 Web 产品。**

MCP Server 是连接外部 Agent（Claude Code / Codex）与 Inkdesk 知识底座的核心通道。先把它做稳，让知识能在对话中自然流转，产品侧的审阅、健康检查、评测再逐步叠加。

### 开发主线

```text
MCP Server（context_pack / deposit / search）
  → 后台编译流水线（异步知识编译 + 审阅队列）
    → 薄仪表盘（审阅 + 健康检查）
      → 完整产品（Ask / Wiki / Skills / Evals）
```

# Inkdesk 知识资产只读联调设计

## 1. 目标

把已经落好的 Spring Boot 知识只读接口真正接入前端，让公共面与主系统里最小可见的知识链路开始读取真实数据库内容，而不是继续完全依赖 `web/lib/mock-data.ts`。

这轮目标是联调底座，不追求把前端所有页面一次性都切到真实后端。

## 2. 范围

- 接入公开首页的文章列表
- 接入公开文章详情页
- 接入 `/app/library` 知识列表
- 接入 `/app/notes/[id]` 知识详情页
- 保留作者资料、项目链接、Agent 建议、计划、搜索推荐等 mock 数据

## 3. 设计

### 3.1 数据访问边界

- 在 `web/lib` 新增面向后端的只读 fetch 层
- fetch 层只负责调用后端接口、处理 HTTP 错误、解析 JSON
- adapter 层负责把后端 DTO 转成现有前端 `types.ts` 中的视图模型
- 页面层只消费前端模型，不直接感知后端 DTO 细节

### 3.2 渐进式兼容策略

- 后端当前 DTO 比前端页面需要的字段更少
- 对缺失但 UI 当前仍依赖的字段，优先用当前 fixture 通过 `note id` 做只读补充
- 这样可以在不扩张后端接口的前提下，先把真实数据库内容接进页面
- 后续如果前端完全摆脱这些补充字段，再推动后端 DTO 扩展

### 3.3 页面接入方式

- 公开首页：从后端读取公开文章列表，作者资料与项目仍来自 fixture
- 公开文章页：正文与标签来自后端，作者信息与继续阅读侧栏继续复用现有前端补充数据
- 知识库页：树接口转成扁平的知识摘要列表，文件夹名由父节点恢复
- 知识详情页：详情接口提供正文和发布状态，其余当前 UI 所需元数据由 adapter 从 fixture 补足

### 3.4 运行时配置

- 前端通过环境变量读取后端地址
- 未配置后端地址时，回退到现有 mock 数据源
- 这样保证本地未启动后端时，前端仍可开发和跑测试

## 4. 不做的事

- 不实现真实登录替换
- 不实现搜索后端
- 不实现计划后端
- 不新增写接口、发布写操作或编辑保存
- 不扩展后端到完整前端视图模型

## 5. 验收标准

- 配置后端地址后，公开首页能展示数据库里的公开文章
- `/articles/[slug]` 能读取真实文章正文
- `/app/library` 能展示真实知识节点列表
- `/app/notes/[id]` 能展示真实正文与发布状态
- 未配置后端地址时，前端继续使用 mock 数据工作

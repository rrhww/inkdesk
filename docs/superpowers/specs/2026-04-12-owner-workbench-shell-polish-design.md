# Inkdesk 主系统壳层与 Agent 控制台打磨设计

## 1. 目标

在不扩张信息架构的前提下，把当前主系统从“已经可用的页面集合”打磨成“主人每天愿意进入的私有工作台中枢”。

这轮只处理主系统，不改公共面定位，不新增真实后端能力。

## 2. 范围

- 强化主系统壳层的工作台感
- 让 Agent 控制台第一屏更像“今日上下文中枢”
- 补上移动端主系统导航，避免当前只有桌面侧边栏的问题

## 3. 具体改动

### 主系统壳层

- 侧边栏增加当前导航高亮
- 顶部标题区增加轻量状态带，说明当前系统模式与今日聚焦
- 增加移动端顶部导航入口，保证手机上也能切换 `Agent / 笔记 / 任务与计划 / 检索 / 发布`

### Agent 控制台

- 第一屏拆成更明确的三层：
  - 今日聚焦与 Agent 主建议
  - 当前任务 / 计划摘要
  - 最近知识资产与行动入口
- 强化“Agent 在组织上下文，而不是只展示卡片列表”的表达
- 增加一个紧凑的“工作台摘要带”，把任务数、待整理知识和可公开内容放在同一视野

### 数据与语义

- 扩展 mock 数据，补充工作台摘要所需字段
- 保持发布模块为次级，不让视觉中心重新偏向发布

## 4. 不做的事

- 不新增新的主导航模块
- 不调整公共首页结构
- 不接真实 Agent、真实任务系统或真实认证

## 5. 验收标准

- 桌面与移动端都能切换主系统核心页面
- `/app` 第一屏能明显看出 `Agent + 任务计划 + 知识资产` 的中枢关系
- 主系统壳层在文案与层级上持续强调“超级个人工作台”而不是普通列表页

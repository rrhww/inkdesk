# Inkvault 双面超级个人工作台设计说明

## 1. 官方定位

- 主定位：超级个人工作台
- 副定位：由个人笔记系统、Agent 中枢和任务 / 计划系统组成的双面系统

## 2. 双面系统模型

Inkvault 固定为两张面孔：

- 公共面：`/` 与 `/articles/[slug]`，对访客开放，只展示作者信息与公开内容
- 主系统：`/login` 与 `/app/*`，只服务主人登录后的私有使用

关键规则：

- 公共面不暴露主系统入口
- 主人通过隐藏入口 `/login` 进入
- 已登录后访问 `/` 自动进入 `/app`

## 3. MVP 三大支柱

- Agent 控制台
- 笔记与知识库
- 任务 / 计划

发布能力保留，但降级为次级模块，用于把部分内容同步到公共面。

## 4. 公共面与主系统边界

### 公共面

- 作者介绍
- 公开文章列表
- 项目 / 链接
- 公开文章详情页

### 主系统

- Agent 首页
- 笔记与知识库
- 知识资产编辑器
- 任务 / 计划
- 全局检索 / 知识召回
- 次级发布模块

## 5. 设计与实现原则

- 不把公共面做成产品官网
- 不把主系统继续做成以发布为中心的旧式管理界面
- Agent 必须成为主人进入系统后的第一屏
- 笔记继续作为底层事实来源
- 任务 / 计划必须占住未来超级工作台的主入口

## 6. 这轮改造范围

- 全面重写产品、设计、架构、交付文档中的旧定位
- 让前端首页、导航、主系统首页和关键页面改成新模型
- 通过 mock owner auth + Cookie 验证“公共面 / 主系统”的入口规则

## 7. 这轮暂不做的事

- 真实后端认证
- 完整任务系统后端
- 真正的 Agent 执行能力
- 自动化编排引擎

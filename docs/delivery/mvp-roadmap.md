# MVP 路线图

## 总目标

把 Inkdesk 做成一个以主系统为核心、同时具备公开输出能力的 MVP。

## Phase 0：文档与产品模型

- 完成重定位文档
- 固定主系统 + 输出层结构

## Phase 1：公开输出与入口骨架

- 公开输出首页
- 公开文章页
- 隐藏登录入口
- 根路由切换逻辑

## Phase 2：主系统核心骨架

- Agent 控制台
- 笔记与知识库
- 任务 / 计划
- 全局检索
- 次级发布模块

## Phase 3：真实数据接入

- 真实认证
- 真实笔记数据
- 真实计划数据
- 真实发布链路

## Phase 4：质量与联调

- 主系统私有访问验证
- 公开输出访问验证
- 主系统与公开输出发布验证

## Phase 5：上线

- 公开输出入口可访问
- 主系统可登录
- 公开文章可阅读

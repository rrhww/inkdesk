# 详细开发计划

## 目标

将 Inkdesk 作为“主系统优先、公开输出从属”的个人系统推进到一个可稳定使用的 MVP。

## 项目边界

- 产品形态：公开输出入口 + 私有主系统
- 主系统核心：Agent 控制台、笔记系统、任务 / 计划、检索
- 输出层核心：作者介绍、公开文章、项目 / 链接
- 技术栈：`Next.js + Spring Boot + PostgreSQL + COS + MinIO`

## 阶段顺序

### Phase 1：产品重定位与前端入口改造

- 重写核心文档
- 重构 `/`、`/login`、`/app`
- 完成公开输出入口与主系统入口分离

### Phase 2：主系统骨架成型

- 完成 Agent 控制台
- 完成笔记与知识库
- 完成任务 / 计划
- 完成检索页
- 保留次级发布模块

### Phase 3：真实接口接入

- 接入真实认证
- 接入笔记、计划、检索、发布接口
- 替换前端 mock 数据

### Phase 4：质量收口

- 测试主系统与公开输出入口
- 测试公开文章链路
- 测试未登录访问私有路由拦截

### Phase 5：部署与上线

- 部署公开输出入口与主系统
- 完成域名、对象存储、Nginx、备份

## 当前开发优先级

1. 公开输出首页与主人入口
2. Agent 控制台
3. 任务 / 计划
4. 笔记与知识库
5. 检索
6. 发布模块
7. 真实后端接入

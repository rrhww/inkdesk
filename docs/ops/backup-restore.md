# 备份与恢复

## 目标

这份文档用于明确 Inkvault MVP 的数据保护策略，保证即使服务器故障或误操作，也能恢复到可继续使用的状态。

## 备份范围

MVP 需要保护的核心资产包括：

- PostgreSQL 数据库
- vault 中的 `raw/` 与 `wiki/` markdown
- 对象存储中的图片资源
- 部署配置与环境变量说明

## 一、数据库备份

### 备份方式

- 使用 `pg_dump` 进行逻辑备份
- 备份文件定时保存到对象存储或服务器安全目录

### 备份频率

- 每日一次全量备份

### 保留策略

- 保留最近 7 天日备份
- 保留最近 4 周周备份

## 二、对象存储备份

### 备份对象

- wiki 页面引用图片
- 研究材料相关图片资源

### 策略

- 优先使用对象存储自身的冗余与生命周期能力
- 如有必要，可定期同步资源清单

## 三、恢复目标

MVP 的最小恢复目标为：

- 恢复到最近一次成功备份的数据库状态
- 恢复 vault 中的 raw 与 wiki 内容
- 恢复主系统可登录
- 恢复已上传图片可被引用

## 四、数据库恢复步骤

### 标准步骤

1. 准备新数据库实例或清空目标库
2. 导入最近的可用备份
3. 校验表结构和数据完整性
4. 启动应用并验证登录、raw / ingest / wiki / ask 是否可用

## 五、对象存储恢复步骤

### 标准步骤

1. 确认桶和访问策略存在
2. 恢复缺失对象或重新同步对象文件
3. 抽样验证 wiki 或研究材料中的图片链接

## 六、恢复演练要求

- 首次正式上线前至少完成一次完整恢复演练
- 演练结果要记录恢复耗时和失败点
- 只有演练通过后，备份策略才算真正成立

## 七、风险边界

当前 MVP 备份策略仍有以下边界：

- 不保证秒级恢复
- 不保证跨地域容灾
- 不保证对象存储版本控制已启用

对于当前单人私有研究系统，这个强度足够；后续如果引入更重的自动化或更高可用要求，再升级。

# Stitch 提示词

## 通用背景

产品名称：Inkvault  
官方定位：围绕长期项目运转的个人主系统  
当前实现形态：公开输出入口 + 主系统

产品模型：

- 对外是公开文章与项目分享
- 对内是主人登录后进入的主系统

统一禁用项：

- 不要做成企业 SaaS 管理台
- 不要加入多人协作、审批流、团队成员、消息中心
- 不要做成产品官网式强营销落地页
- 不要暴露公开输出页中的主系统入口

## 1. 公共首页 `/`

请为 Inkvault 设计“公开输出首页 / 分享层”。

页面目标：

- 让访客理解作者是谁
- 看到公开文章
- 看到项目 / 链接入口

必须模块：

- 作者介绍
- 公开文章列表
- 项目 / 链接

视觉关键词：

- 个人分享页
- 长期主义
- 安静但不无聊
- 不像产品官网

## 2. Agent 控制台 `/app`

请为 Inkvault 设计主人登录后看到的首页，它必须是 `Agent 控制台`。

页面目标：

- 让 Agent 成为第一屏
- 展示今日上下文
- 同时给出任务和笔记入口

必须模块：

- Agent 建议卡片
- 今日任务摘要
- 最近笔记入口

视觉关键词：

- 中枢
- 判断
- 协同
- 不像传统 dashboard

## 3. 知识资产编辑器 `/app/notes/[id]`

请为 Inkvault 设计“知识资产编辑器”。

页面目标：

- 表达这是主系统里的知识资产，而不是普通管理编辑页
- 展示发布状态与公开输出入口

必须模块：

- 标题区
- 正文画布
- 状态标签
- 侧边元信息

## 4. 任务与计划 `/app/plans`

请为 Inkvault 设计“任务与计划”页面。

页面目标：

- 表达主系统不只是记录，而是推进事情

必须模块：

- 今日任务
- 进行中计划
- 状态标签

## 5. 笔记与知识库 `/app/library`

请为 Inkvault 设计“笔记与知识库”页面。

页面目标：

- 展示长期知识资产
- 体现这些内容是 Agent 与主系统的事实来源

必须模块：

- 列表
- 标签
- 更新时间
- 归属状态

## 6. 全局检索 `/app/search`

请为 Inkvault 设计“全局检索 / 知识召回”页面。

页面目标：

- 表达检索不是找文件，而是重新召回上下文

必须模块：

- 搜索框
- 结果列表
- 空状态提示

## 7. 发布模块 `/app/publish`

请为 Inkvault 设计“发布模块”。

页面目标：

- 把主系统里的成熟内容发布到公开输出层
- 明确它是次级模块，而不是产品中心

必须模块：

- 已发布列表
- 未发布列表

## 8. 公开文章页 `/articles/[slug]`

请为 Inkvault 设计公开文章页。

页面目标：

- 稳定承接阅读
- 保持和公开输出首页一致的作者气质

必须模块：

- 标题
- 正文
- 返回公开输出首页入口
- 作者摘要

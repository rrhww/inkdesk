# 设计工作流

## 目标

确保 Inkvault 的设计工作始终围绕“私有研究闭环”展开，而不是回到已废弃的旧路线。

## 当前设计顺序

1. 先定 `问答`
2. 再定 `资料 / 审阅 / 知识库`
3. 最后补登录、空状态和异常状态

## 当前设计原则

- `问答`：像可信的研究对话入口，而不是聊天玩具或企业后台
- `资料`：像原始证据工作台，而不是表格管理页
- `审阅`：像知识确认闸门，而不是模糊的通知中心
- `知识库`：像沉淀后的长期记忆层，而不是普通文档列表
- 全部页面都要强化 `raw -> ingest -> wiki -> ask` 的关系

## 设计交付要求

- 页面命名与路由命名保持一致
- 所有页面说明统一使用 `问答 / 资料 / 审阅 / 知识库` 术语
- 设计稿必须能帮助用户分清原始材料、待确认提案和正式知识三层

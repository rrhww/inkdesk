# 设计工作流

## 目标

确保 Inkdesk 的设计工作始终围绕“主系统优先，公开输出从属”的模型展开，而不是回到旧的“以发布为中心的单面产品”思路。

## 当前设计顺序

1. 先定公开输出首页与 Agent 控制台
2. 再定知识资产编辑器与任务计划
3. 最后补检索、发布模块与公开文章页

## 当前设计原则

- 公开输出层：像稳定的个人分享页，而不是产品官网
- 主系统：像私有中枢，不像传统企业管理界面
- Agent：首页第一屏必须成立
- 任务 / 计划：必须占住核心模块位置
- 发布：只能是次级模块

## 设计交付要求

- 页面命名与路由命名保持一致
- 所有页面说明统一使用“公开输出 / 主系统”术语
- 设计稿不能暴露公开输出页中的系统登录入口

# HTML 方向复核

## 结论

旧方向中“公开输出阅读体验”和“主系统首页”分裂得太明显，不再适合当前的主系统优先定位。

现在的评审标准改为：

- 公开输出层必须像稳定的个人分享页
- 主系统必须像私有中枢，而不是传统管理界面
- Agent 控制台必须成为主人登录后的第一印象

## 当前明确要求

### 公开输出首页必须像什么

- 作者介绍页
- 公开文章入口
- 长期项目与链接聚合

### 公开输出首页不能像什么

- 营销 landing page
- 带登录 CTA 的产品首页

### 主系统首页必须像什么

- Agent 中枢
- 今日上下文入口
- 任务与笔记的汇聚页

### 主系统首页不能像什么

- 企业管理 dashboard
- 普通文档列表页

## 后续动作

- 所有 HTML / Stitch 方向都按“主系统 + 输出层”复核
- 不再沿用“以发布为中心”的旧判断标准

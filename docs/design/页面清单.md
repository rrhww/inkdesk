# 页面清单

## 目标

逐页说明 AI 研发自动化控制台需要承载的页面目标和状态。

## 页面清单

| 页面 | 路由 | 主要目标 | 关键模块 | 关键状态 | 优先级 |
| --- | --- | --- | --- | --- | --- |
| 根路由分流 | `/` | 按登录态重定向 | session 检查、重定向 | 未登录、已登录 | P0 |
| 登录页 | `/login` | 进入私有工作区 | 登录表单、错误提示 | 默认、失败 | P0 |
| 首页 | `/app` | 展示 Dev Run Console 与研发任务入口 | 新建任务、运行中任务、阶段轨道、待确认输出、知识健康摘要、可用 Skills | 空状态、有任务、待确认、运行中 | P0 |
| Context Ask 页 | `/app/ask` | 为当前研发任务发起上下文查询与追问 | 提问表单、回答、知识缺口、follow-up、writeback / deposit | 空状态、有回答、联网补料、追问中 | P0 |
| Raw 页 | `/app/raw` | 管理原始材料和任务证据 | 来源列表、PRD 导入、网页导入、文本导入、PDF 导入 | 空状态、有来源、导入中 | P0 |
| Ingest 页 | `/app/ingest` | 审阅 AI 提案与阶段沉淀 | 提案列表、来源、topic 归属、阶段归属、接受/拒绝 | 有提案、提案已处理 | P0 |
| Wiki 列表页 | `/app/wiki` | 浏览知识页和 claim | topic 列表、摘要、来源数、claim 健康 | 空状态、有结果 | P0 |
| Wiki 详情页 | `/app/wiki/[id]` | 阅读单个知识页 | current understanding、key claims、open questions、sources、research thread | 正常、主题不存在 | P0 |
| Runs 页 | `/app/runs` | 查看研发任务历史 | run 列表、阶段状态、输出摘要、阻塞原因 | 规划中 | P1 |
| Skills 页 | `/app/skills` | 浏览研发技能 | 技术方案、技术评审、coding、测试准备、测试修复、问题排查 | 规划中 | P1 |
| Health 页 | `/app/health` | 查看知识与自动化健康 | wiki health、skill health、eval signals、run blockers | 规划中 | P1 |

## 兼容页面

以下页面不再作为独立产品面，只保留跳转：

- `/app/inbox`
- `/app/review`
- `/app/topics`
- `/app/sources`

## 页面状态要求

- 登录页覆盖默认和失败状态
- 首页至少覆盖无任务、有运行中任务、待确认阶段输出三种状态
- Raw 页覆盖导入表单与已有来源列表
- Ingest 页覆盖待审阅和已处理反馈
- Wiki 详情页覆盖 understanding / claims / questions / sources
- Context Ask 页覆盖空状态、有答案、continueFromAskTurnId、`vault_plus_web`

## 实现说明

- 当前没有公开首页和公开文章页
- 当前没有 `plans`、`search`、`publish`、`settings` 页面
- 页面文案应统一使用 `run / context / raw / ingest / wiki / skill / eval` 语言
- `/app` 不应继续表达为 Ask-first；Ask 是 Context Ask，不是产品首页

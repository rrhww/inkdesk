# 原型计划

## 目标

这份文档用于安排当前重定位后的原型输出顺序，确保设计资源优先投入在真正影响产品转向的页面上。

## 当前原型主题

- 主题 A：公开输出首页 / 分享层
- 主题 B：Agent 控制台
- 主题 C：知识资产编辑器
- 主题 D：任务与计划

## 首批必须完成的页面

1. 公共首页 `/`
2. 登录页 `/login`
3. Agent 控制台 `/app`
4. 笔记与知识库 `/app/library`
5. 知识资产编辑器 `/app/notes/[id]`
6. 任务与计划 `/app/plans`
7. 全局检索 `/app/search`
8. 公开文章页 `/articles/[slug]`

## 输出顺序

### 第一轮：定产品气质

- 公开输出首页
- Agent 控制台

### 第二轮：定主系统主线

- 知识资产编辑器
- 任务与计划
- 笔记与知识库

### 第三轮：补完整闭环

- 全局检索
- 登录页
- 发布模块
- 公开文章页

## 验收重点

- 公开输出层不能像产品官网
- 主系统不能像传统企业管理界面
- Agent 控制台必须有“第一屏中枢感”
- 任务与计划必须真实占住主入口，而不是后加的小页面

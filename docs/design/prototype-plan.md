# 原型计划

## 目标

这份文档用于安排当前研究工作台的原型输出顺序，确保设计资源优先投入在真正影响主闭环的页面上。

## 当前原型主题

- 主题 A：问答
- 主题 B：资料
- 主题 C：审阅
- 主题 D：知识库

## 首批必须完成的页面

1. 登录页 `/login`
2. 问答 `/app`
3. 资料 `/app/raw`
4. 审阅 `/app/ingest`
5. 知识库 `/app/wiki`
6. 知识页详情 `/app/wiki/[id]`

## 输出顺序

### 第一轮：定产品第一感知

- 问答
- 登录页

### 第二轮：定研究闭环结构

- 资料
- 审阅
- 知识库

### 第三轮：补完整体验

- 知识页详情
- 空状态
- 错误状态

## 验收重点

- 问答必须像研究入口，而不是后台首页
- 资料、审阅、知识库三层必须界限清楚
- 整体必须让用户快速理解 `raw -> ingest -> wiki -> ask`

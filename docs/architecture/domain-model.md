# 领域模型

## 目标

定义当前 MVP 需要稳定下来的核心实体，为主系统与公开输出层的实现提供统一语言。

## 核心角色

### Owner

系统主人。

职责：

- 登录隐藏入口
- 进入主系统
- 维护笔记与计划
- 决定哪些内容进入公开输出层

### Visitor

公开输出访客。

职责：

- 阅读公开文章
- 浏览作者信息与项目链接

## 核心实体

### NoteDocument

主系统中的知识资产。

字段关注点：

- 标题
- 摘要
- Markdown 正文
- 标签
- 更新时间
- 是否发布到公开输出层

### PlanItem

任务 / 计划项。

字段关注点：

- 标题
- 摘要
- 状态
- 时间范围
- 与笔记的潜在关系

### Publication

知识资产进入公开输出层的发布记录。

字段关注点：

- 对应笔记
- slug
- 是否公开
- 最近发布时间

### Asset

主系统中的图片资源。

字段关注点：

- 所属笔记
- 资源地址
- 对象存储位置

## 派生视图

以下内容在 MVP 中可以先作为前端或服务层派生视图，不要求一开始独立建模为复杂实体：

- Agent 建议
- 今日上下文摘要
- 公开输出首页聚合视图

## 边界说明

- Public Homepage 不是独立业务实体，而是由 Owner 信息、公开文章和项目链接聚合出的视图
- Agent 控制台在 MVP 里先是聚合中枢，不要求一开始有完整智能代理生命周期

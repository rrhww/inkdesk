# 领域模型

## 目标

统一当前 vault-first AI R&D Automation Runtime 的业务语言，避免再混用旧的 `notes / plans / publish / public` 术语。

当前目标用户主路径是 `PRD / 研发任务 -> Dev Run -> Review / Deposit`。`raw / ingest / wiki / ask / claim / health` 是系统内部协议，除必要裁决外不应强迫用户日常理解。Ask 是 Context Ask，服务当前研发任务，不是产品首页。

## 核心角色

### Owner

当前唯一主要用户。

职责：

- 登录私有工作区
- 导入原始材料到 `raw/`
- 创建 PRD / bug / 改造任务
- 在 Dev Run 中发起 Context Ask、追问、决定是否沉淀回答或阶段输出
- 在必要时审阅 AI 提案或裁决冲突
- 维护最终进入 `wiki/` 的长期记忆边界

### ExternalAgent

未来通过 MCP / CLI / Skill 接入 Inkdesk 的外部 Agent。

典型来源：

- Claude Code
- Codex
- Cursor

职责：

- 任务前向 Inkdesk 请求 context pack
- 任务后在用户确认下提交方案、代码决策、测试修复或复盘的 deposit 请求
- 不直接写入 `wiki/` 或 vault 长期真相

## 核心实体

### Workspace

单 owner 工作区。

职责：

- 承载 owner 的所有研究数据
- 作为 source、topic、review、ask 的逻辑归属边界

### Source

原始材料，也就是 `raw/` 中的来源对象。

典型来源：

- 网页
- PDF
- 手动文本
- 从 legacy note 迁移出的历史内容

关键语义：

- 先进入 `raw`
- 可以被编译进一个或多个 topic
- 必须保留可回溯定位信息

### ReviewItem

由 AI 生成、等待人工决策的提案。

关键语义：

- 是 `ingest` 页面上的主对象，也是后台沉淀失败或需要裁决时的落点
- 可以创建新 topic，也可以 patch 现有 topic
- 未接受前不能直接改写 wiki

### Topic

已沉淀的知识页，也就是 `wiki/` 中的核心对象。

关键语义：

- 有 summary、current understanding、open questions
- 通过 claims 和 sources 保留证据链
- 最终内容可以从 vault Markdown 恢复

### TopicClaim

知识页中的关键论点。

关键语义：

- 对应一句可引用的稳定判断
- 应尽量能追溯到 source

### TopicThreadEntry

知识页的研究过程片段。

关键语义：

- 保存提炼过程、补充说明和研究痕迹
- 帮助 owner 理解当前知识页是如何形成的

### AskTurn

一次 Context Ask 与其上下文快照。

关键语义：

- 支持 topic scoped 和 global ask
- 支持追问链
- 记录引用、知识缺口、follow-up questions 与 deposit/writeback 包
- 是用户点击“沉淀这次回答”时的主要输入对象

### DevRun

一次研发任务运行。

关键语义：

- 来源可以是 PRD、bug、改造目标或外部 Agent 任务
- 记录 Context Pack、技术方案、技术评审、coding、测试、修复、沉淀等阶段状态
- 每个阶段输出都可以进入 review-first 流程
- 是未来 Harness 编排的主对象

### DepositRequest

一次沉淀请求。

关键语义：

- 可以来自 Inkdesk Web App 的回答卡片
- 可以来自外部 Agent 的 MCP / CLI 调用
- 输入包括 question、answer、selection、citations、source_context、repo/task/run 信息
- 由主 Agent 编排专项子 Agent 完成后续处理

### DepositTrace

后台沉淀流水线的执行痕迹。

关键语义：

- 记录 Insight Extractor、Evidence Binder、Topic Router、Conflict Checker、Patch Writer、Quality Gate 的关键输入输出
- 用于调试、评测和后续策略升级
- 不直接面向普通用户展示

### RetrievalChunk

为 Ask 和检索准备的 chunk 级索引对象。

关键语义：

- 是内部检索基础设施，不直接面向 UI
- 可支持 lexical fallback 与 embedding/hybrid retrieval

### LegacyNoteBridge

由 `content_nodes` 和 `note_documents` 组成的迁移兼容层，不是新主产品对象。

关键语义：

- 用于接住旧内容
- 用于初始化 demo 数据和过渡期研究编译

## 派生视图

以下内容目前主要作为服务层或前端聚合视图存在：

- Dev Run Console
- suggestedQuestions
- Ask thread view
- ingest queue summary
- deposit status / conflict confirmation card

## 边界说明

- 当前没有访客角色，也没有公开阅读面
- 当前没有 plans / publish / settings 主路径
- Ask 是 Context Ask 和沉淀入口，不等于自治 Agent 执行系统
- Dev Run 是 AI 研发自动化的用户主入口
- Vault Markdown 是长期真相，数据库和 UI 都要围绕它对齐
- 外部 Agent 只允许通过 context/deposit 协议接入，不直接写长期知识

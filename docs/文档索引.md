# 文档索引

`docs/` 用来解释当前代码已经实现、正在维护、仍值得继续扩展的内容。

当前产品边界已经收敛为：

- 单人私有
- vault-first
- Dev Run-first
- 用户主路径是 `PRD / 研发任务 -> Dev Run -> Review / Deposit`
- 内部闭环是 `raw -> ingest -> wiki -> ask -> health -> ingest`
- 长期目标是作为 Claude Code、Codex、Cursor 等外部 Agent 的研发上下文、沉淀和自动化门禁底座

旧的“公开输出层 / plans / note editor”方向已经退出主路径；相关过时文档已删除或收缩，不再作为项目规范来源。

## 分层说明

- `product/`：当前产品定位、用户故事、阶段路线
- `design/`：当前路由结构、页面清单、视觉令牌
- `architecture/`：系统边界、领域模型、接口、数据库与技术决策
- `delivery/`：本地开发、协作方式、验收与提交流程
- `ops/`：环境变量、部署、备份恢复
- `references/`：外部文章、方法论来源及其与 Inkdesk 的映射
- `resume/`：面试讲述、简历包装和项目表达材料
- `development/`：实现计划、技术设计与知识库设计，作为产品主规范的落地依据

## 文档维护原则

- 文档全部使用中文，接口、路径、命令保留英文
- 同一项事实只保留一个主归属文档，避免复制粘贴式维护
- 当前代码与运行方式优先于历史方案；过时方案应删除或显式归档
- 聊天记录不是规范，真正生效的是仓库里的文档和代码
